# Google Drive 重複案件フォルダ統合ツール

Google Drive 内の案件フォルダ（`番号_店舗名_媒体名` 形式）で重複しているものを検出し、
標準子フォルダ構造を1つに統合するツールです。

---

## 前提

### 対象フォルダ構造

```
番号_店舗名_媒体名/
├── アウトプット/
└── 制作データ/
    ├── 入稿前データ/
    └── 入稿データ/
```

同一の `店舗名_媒体名` を持つフォルダが複数ある場合を「重複」と判定します。

### 統合ルール

| 状況 | 処理 |
|------|------|
| 正規フォルダに同名ファイルなし | 正規フォルダへ移動 |
| 名前・内容が完全一致（md5 / size / 更新日時） | スキップ（削除しない） |
| 名前同一・内容が異なる | `要確認_重複/` フォルダへ退避 |

**削除は一切行いません。**
**正規フォルダ = 更新日時が最新のフォルダ**

## 起動方法（Web UI）

```bash
cd gdrive-merger
npm install
npm run auth   # 初回のみ（token.json を生成）
npm run dev    # http://localhost:3000 を開く
```

ブラウザで `http://localhost:3000` にアクセスし、以下の手順で操作します:

1. **起点フォルダ ID** を入力（空欄 = My Drive 直下）
2. **「重複をスキャン」** → 重複グループ一覧を確認
3. **「dry-run で確認」** → 移動・退避の予定一覧を確認
4. **「整理を実行」** → 確認ダイアログ後に実際に移動（dry-run 後にボタンが有効化）

## セットアップ（詳細）

### 1. リポジトリを準備

```bash
cd gdrive-merger
npm install
```

### 2. Google Cloud Console で認証情報を作成

1. [Google Cloud Console](https://console.cloud.google.com/) を開く
2. プロジェクトを作成（または既存を選択）
3. **API とサービス → ライブラリ** で `Google Drive API` を有効化
4. **API とサービス → 認証情報** から
   「認証情報を作成」→「OAuth クライアント ID」→「デスクトップ アプリ」
5. JSON をダウンロードして `gdrive-merger/credentials.json` として保存

### 3. 初回認証（token.json の生成）

```bash
npm run auth
```

ブラウザで Google アカウントにログインし、表示されたコードをターミナルに貼り付けます。
`token.json` が生成されれば完了です。

## CLI での使い方

Web UI を使わず CLI のみで実行することも可能です。

### Dry-run（計画確認のみ・推奨）

```bash
node index.js --dry-run [--root <フォルダID>]
# または
npm run dry-run
```

### 本実行（ファイル移動）

```bash
node index.js --apply [--root <フォルダID>]
# または
npm run apply
```

フォルダ ID は Drive URL の末尾 `…/folders/<ここ>` から確認できます。

## ログレベル

`LOG_LEVEL` 環境変数で詳細度を変更できます（デフォルト: `INFO`）。

```bash
LOG_LEVEL=DEBUG node index.js --dry-run
```

| レベル | 内容 |
|--------|------|
| DEBUG | 全ファイルの比較結果を表示 |
| INFO | 移動・スキップ・退避のみ表示（デフォルト） |
| WARN | 警告・退避のみ |
| ERROR | エラーのみ |

## 出力レポート

`reports/` ディレクトリに以下が生成されます:

| ファイル | 内容 |
|---------|------|
| `dry-run_<日時>.csv` | アクション計画（Excel 対応 BOM 付き UTF-8） |
| `dry-run_<日時>.json` | 同上（機械処理向け） |
| `apply_<日時>.csv` | 実行結果 |
| `apply_<日時>.json` | 同上 |

### CSV の列

| 列名 | 説明 |
|------|------|
| タイプ | `MOVE` / `SKIP_EXACT` / `QUARANTINE` |
| ファイル名 | 対象ファイル名 |
| ファイルID | Google Drive ファイル ID |
| 移動元フォルダパス | `案件フォルダ名/アウトプット` など |
| 移動元フォルダID | Google Drive フォルダ ID |
| 移動先フォルダパス | 移動先（スキップの場合は空） |
| 移動先フォルダID | 移動先 ID（スキップの場合は空） |
| 理由 | 処理理由の詳細 |

---

## 注意事項

- **本ツールはファイルを削除しません。** 統合元ファイルは移動のみです。
- `要確認_重複` フォルダは正規フォルダの各子フォルダ内に作成されます。
- `token.json` と `credentials.json` は `.gitignore` で除外されています。
  絶対にリポジトリにコミットしないでください。
- 大量のファイルを処理する場合、Drive API のレート制限に注意してください。
