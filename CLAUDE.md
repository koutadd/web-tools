# AICARE web-tools — 開発ポリシー（保守最優先モード）

## 基準版
- タグ: `ui-baseline-v1` / `app-baseline-v1`
- 現在の UI・レイアウト・導線・見た目を「基準版」として固定する
- 全ての開発はこの基準版に**追加で積み上げる**こと

## 絶対ルール（全面改修禁止）
- 既存ページ（*.html）の全面再設計・全面置き換えは禁止
- 既存 UI・レイアウト・共通ナビ・スタイルの大幅変更は禁止
- 「デザイン改善」「UI刷新」より「基準版維持」を優先する

## 許可される変更（差分追加が原則）
- 新規ページの追加（新しい *.html ファイル）
- 既存ページへの機能の差し込み追加（既存構造を壊さない範囲）
- バグ修正（最小限の変更で）
- 新規コンポーネント・新規セクションの追加

## 変更前の必須確認
- 編集対象ファイル一覧を必ず出すこと
- 既存 UI に影響がないか確認してから実行すること
- 迷ったら「新規ファイルで対応できないか」を先に検討すること

## UI を崩してよい条件（例外）
- ユーザーが明示的に「全面改修OK」「UIを作り直して」と指示した場合のみ
- それ以外は「既存UIを崩しません」が前提

## ブランチ運用
- main: 安定版・基準版
- 大きな変更は feature/xxx ブランチで作業 → 確認後マージ
- main で直接大改修しない

## LocalStorage キー（参照用）
- aicare-stores.html: aicare_stores_v1, aicare_store_tasks_v1 など
- aicare-lab-app.html: aicare_rows_v3
- aicare-projects.html: 独自キー

## デプロイ
- Vercel (web-tools-black-iota.vercel.app)
- `vercel --prod` で本番デプロイ
